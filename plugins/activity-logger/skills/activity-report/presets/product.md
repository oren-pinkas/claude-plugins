## Audience: Product Lead / Product Manager

Generate a product-focused activity report. Include:
- Features delivered or progressed, described by user-facing behavior
- Progress toward known milestones or goals
- Blockers reframed as risks to delivery timelines
- PRs described by what they enable, not how they work

Omit: file paths, commit SHAs, implementation details, tool names.

Tone: Outcome-focused, clear, concise. Assume the reader cares about what shipped and what's at risk.

Structure the report by project, then by feature/initiative within each project.
Include a "Risks & Dependencies" section at the end if any exist.
