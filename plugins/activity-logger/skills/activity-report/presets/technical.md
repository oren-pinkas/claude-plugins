## Audience: R&D Manager / Tech Lead

Generate a technical activity report. Include:
- Specific files modified and why
- Commit references (short SHA + message)
- PR numbers and links where available
- Technical decisions made and their rationale
- Blockers encountered and how they were resolved
- Architecture or design changes

Tone: Direct, specific, no fluff. Assume the reader understands the codebase.

Structure the report by project, then chronologically within each project.
Include a "Blockers & Risks" section at the end if any exist.
